#### 3.1.2. Описание компонентов графического интерфейса

> _Заполняет: Михалёв Кирилл Денисович_

Главное окно (`MainWindow`) состоит из:

1. **Поле ввода выражения** (`QLineEdit`) — пользователь видит набираемое выражение, может править его как мышью, так и клавиатурой.
2. **Поле результата** (`QLabel`) — отображает либо численный результат (зелёным), либо текст ошибки (красным).
3. **Сетка кнопок** (`QGridLayout`, 5 строк × 5 столбцов):

| Ряд | Кол. 1 | Кол. 2 | Кол. 3 | Кол. 4 | Кол. 5 |
|---|---|---|---|---|---|
| 0 | C | ⌫ | ( | ) | / |
| 1 | 7 | 8 | 9 | * | ln |
| 2 | 4 | 5 | 6 | − | exp |
| 3 | 1 | 2 | 3 | + | sqrt |
| 4 | 0 | . | = | = | abs |

4. **Обработка клавиатуры** (`keyPressEvent`) — `Enter`/`=` запускают вычисление, `Backspace` стирает символ, `Esc` очищает.

#### 3.1.3. Архитектура программы

```
                  ┌─────────────────┐
                  │   MainWindow    │   ← Qt-виджеты, обработка кликов
                  │     (GUI)       │
                  └────────┬────────┘
                           │ строка-выражение
                           ▼
                  ┌─────────────────┐
                  │     Parser      │   ← токенизация + Shunting-yard
                  │ tokenize, to_rpn │
                  └────────┬────────┘
                           │ массив токенов в RPN
                           ▼
                  ┌─────────────────┐
                  │    Evaluator    │   ← стековое вычисление + проверки
                  │    evaluate     │
                  └────────┬────────┘
                           │ double / исключение
                           ▼
                  ┌─────────────────┐
                  │   QLabel result │
                  └─────────────────┘
```

### 3.2. Разработка

#### 3.2.1. Парсер выражений

> _Заполняет: Загорный Тимофей Максимович_

Парсер реализован в файлах `src/parser.h` и `src/parser.cpp`. Состоит из двух методов:

**1. `tokenize(expression)`** — преобразует строку в массив токенов.

Поддерживаемые типы токенов:
- `Number` — число (целое или с десятичной точкой)
- `Operator` — `+`, `-`, `*`, `/`, унарный минус `u-`
- `Function` — `ln`, `exp`, `sqrt`, `abs`
- `LeftParen`, `RightParen` — скобки

**Унарный минус** определяется по контексту: если `-` стоит в начале выражения, после оператора, после `(` или после имени функции — это унарный минус (`u-`), иначе — бинарное вычитание.

**2. `to_rpn(tokens)`** — преобразование в обратную польскую запись по алгоритму Дейкстры (Shunting-yard).

Таблица приоритетов:
| Оператор | Приоритет | Ассоциативность |
|---|---|---|
| `u-` | 4 | правая |
| `*`, `/` | 3 | левая |
| `+`, `-` | 2 | левая |

**Пример работы:**

Вход: `2+3*4`
После токенизации: `[2] [+] [3] [*] [4]`
После Shunting-yard: `[2] [3] [4] [*] [+]`

Вход: `sqrt(16)+abs(-3)`
После токенизации: `[sqrt] [(] [16] [)] [+] [abs] [(] [u-] [3] [)]`
После Shunting-yard: `[16] [sqrt] [3] [u-] [abs] [+]`

> 📊 **Блок-схема `tokenize`** — см. `screenshots/flowchart_tokenize.png`

#### 3.2.2. Вычислитель RPN

> _Заполняет: Калинин Кирилл Евгеньевич_

Вычислитель реализован в `src/evaluator.cpp`. Алгоритм классический стековый:

```
для каждого токена в RPN:
    если число           → положить на стек
    если бинарный оп.    → снять 2 значения, применить, положить результат
    если унарный минус   → снять 1 значение, изменить знак, положить
    если функция         → снять 1 значение, применить функцию, положить
по окончании: на стеке должно остаться ровно 1 значение
```

**Пример: вычисление `[2] [3] [4] [*] [+]`:**

| Шаг | Токен | Стек до | Действие | Стек после |
|---|---|---|---|---|
| 1 | 2 | `[]` | push | `[2]` |
| 2 | 3 | `[2]` | push | `[2, 3]` |
| 3 | 4 | `[2, 3]` | push | `[2, 3, 4]` |
| 4 | `*` | `[2, 3, 4]` | 3*4 | `[2, 12]` |
| 5 | `+` | `[2, 12]` | 2+12 | `[14]` |

Результат: **14**.

> 📊 **Блок-схема `evaluate`** — см. `screenshots/flowchart_evaluate.png`

#### 3.2.3. Обработка ошибок

> _Заполняет: Калинин Кирилл Евгеньевич_

Вся ошибочная логика выбрасывает исключения двух типов:
- `ParseError` — на этапе разбора выражения (неверный синтаксис),
- `EvalError` — на этапе вычисления (математические ошибки).

В `MainWindow::compute()` оба типа ловятся одним `catch (const std::exception& e)` и выводятся в поле результата красным цветом.

| Класс ошибки | Пример входа | Сообщение |
|---|---|---|
| Деление на ноль | `1/0` | «Деление на ноль» |
| ln от ≤0 | `ln(0)`, `ln(-2)` | «ln от неположительного числа» |
| sqrt от <0 | `sqrt(-4)` | «sqrt от отрицательного числа» |
| Двойной оператор | `2++3` | «Двойной оператор: ++» |
| Несбалансированные скобки | `(2+3` | «Несбалансированные скобки: лишняя '('» |
| Лишняя ')' | `2+3)` | «Несбалансированные скобки: лишняя ')'» |
| Пустое выражение | `` (пусто) | «Пустое выражение» |
| Неизвестная функция | `foo(2)` | «Неизвестная функция: foo» |
| Две точки в числе | `2..3` | «Некорректное число: две точки» |
| Недостаточно операндов | `2+` | «Недостаточно операндов для +» |
| Переполнение | `exp(1000)` | «Переполнение» |

#### 3.2.4. Связка GUI с парсером

> _Заполняет: Михалёв Кирилл Денисович_

Каждая кнопка цифры/оператора через `connect()` привязана к слоту, который дописывает символ в `QLineEdit`. Кнопка `=` (или клавиша Enter) вызывает `compute()`:

```cpp
void MainWindow::compute() {
    QString expr = expression_edit_->text();
    try {
        Evaluator ev;
        double r = ev.evaluate(expr.toStdString());
        result_label_->setText(QString::number(r, 'g', 12));
    } catch (const std::exception& e) {
        result_label_->setText(QString("Ошибка: %1").arg(e.what()));
    }
}
```

Кнопки функций (`ln`, `exp`, `sqrt`, `abs`) добавляют в выражение `имя(`, чтобы пользователь сразу мог писать аргумент.
